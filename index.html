矩阵置零

给定一个 m x n 的矩阵，如果一个元素为 0，则将其所在行和列的所有元素都设为 0。请使用原地算法。

示例 1:

输入: 
[
[1,1,1],
[1,0,1],
[1,1,1]
]
输出: 
[
[1,0,1],
[0,0,0],
[1,0,1]
]

示例 2:

输入: 
[
[0,1,2,0],
[3,4,5,2],
[1,3,1,5]
]
输出: 
[
[0,0,0,0],
[0,4,5,0],
[0,3,1,0]
]

进阶:

一个直接的解决方案是使用  O(mn) 的额外空间，但这并不是一个好的解决方案。
一个简单的改进方案是使用 O(m + n) 的额外空间，但这仍然不是最好的解决方案。
你能想出一个常数空间的解决方案吗？

方案一，直接先拷贝数组，然后检查新的数组，如果某一个元素为零，则将原来的数组相应的行和列都置零。

思考：空间复杂度o(mn),时间复杂度为o(mn x mn)，因为每次检查如果元素为零，都会将当前元素的行和列置零。
改进：能不能只重置一次。

方案二 只记录了哪些行和列需要置零，然后将相应的行和列置零就行了

思考：空间复杂度为o(m+n),时间复杂度o(mn),依然不符合题目的要求，怎样用常熟级空间呢？

方案三: 使用现有的空间，如果某一行为零，则行首置零，某一列为零，列首置零，但是(0,0)位置代表行首，则需要额外的空间代表列，所以使用常数空间就解决问题。

思考:只使用了额外变量tmp,空间复杂度为o(1),时间复杂度为o(mn)

总结：

算法题的核心思路就是空间换时间，时间换空间。最优的就是最少时间，最小空间解决问题。

相交链表

相交链表
编写一个程序，找到两个单链表相交的起始节点。

方法一 暴力法

遍历法，常规思路，遍历两遍。很明显，中级以上难度的题目对时间复杂度有要求，行不通。$ O(n^2) $ 的时间复杂度，判定超时，放弃。

方法二 hash表

既然用暴力的两遍遍历的方法不行，那就牺牲空间换时间，遍历链表，将节点存放到一个set中，如果有重复的节点，直接返回。时间复杂度$O(n)$,空间复杂度$O(n)$。

方法三

优化方法

设有两个链表，l1和l2，长度分别是len1和len2，如果有交点，相交的长度是x。则链表len1-len2的长度为len3。如果两个链表相交，则从头遍历到链表尾部，如果相同则说明链表相交。然后，再把长的链表移动到len3的位置，然后开始同时向前移动链表头结点，如果某一个节点两个链表的节点相同，则说明到了相交的节点。


优化后，时间复杂度变成了$O(n)$,但是空间使用还是很高，空间复杂度明明是O(1)呀，看来是代码问题了，今天时间有限，明天再优化ヾ(≧▽≦*)o！
